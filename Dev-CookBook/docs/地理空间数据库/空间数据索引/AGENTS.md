# 地理空间数据索引整理规范

## 范围与来源

- 本目录仅整理《地理空间数据库原理（第二版）》第四章“地理空间数据索引”。
- 资料用于个人学习记录。原书著作权及相关权利归作者崔铁军及权利人所有。


## 整理规则

- 按原书的章节顺序、标题层级、段落顺序和编号整理，正文不得遗漏、删改或调换。
- 每个小节文件从本节标题开始，至下一节标题前结束；保留原有的图号、表号、公式号、示例号和参考编号。
- 一个小节对应一个独立文件夹，文件夹内使用 `index.md` 存放正文，并将该小节使用的图片、表格截图等资源放在同级 `assets/` 中；例如 `4.1-空间索引概述/index.md` 与 `4.1-空间索引概述/assets/`。
- 顶层 `index.md` 只做章节导航；顶层目录不集中存放所有小节图片资源，避免后续资源混杂。
- 受 VitePress 页面结构限制，正文 Markdown 标题从二级标题开始，不使用一级标题；例如 `## 4.1.1 空间索引的发展`、`## 4.1.2 空间索引分类`。
- 每个小节已经独立成一个 Markdown 文件时，文件内不再使用章标题或本节标题占用标题层级；章标题、本节标题如需保留，可作为普通加粗文本。
- 原文中的编号分类可以改造成 Markdown 标题以增强索引性，例如 `1. 按搜索对象分类` 可整理为三级标题，`1）对象分割法` 可整理为四级标题；标题文字保留原编号和原意。
- 图、表及其他版面资源不在正文中额外重复完整图题或表题；在原文出现资源引用的位置加粗编号标记即可，例如 `**（图 4.13）**`、`**表 4.1**`。
- 图表加粗引用前后需要保留空格，避免 VitePress/Markdown 解析器将星号按普通文本渲染；例如写成 `提出 **（图 4.13）**。`，不要写成 `提出**（图 4.13）**。`。
- 资源截图紧跟在对应引用段落之后，Markdown 图片链接的 `alt` 文本可以保留完整图题或表题，便于无障碍阅读和维护。
- 可以可靠转写的表格使用 Markdown 表格，并保留原表号和表题；无法可靠转写时保留原编号、表题及截图引用。
- 原书代码使用 fenced code block 包装；无法判断语言时使用 `text`。
- 原书数学公式使用 Markdown 数学公式包装：行内公式使用 `$...$`，独立公式使用 `$$...$$`；保留原公式编号。
- 截图必须清晰、完整，不裁掉图题、表题、坐标、图例、注释或必要的上下文。
- 图表截图必须以单张图或单个表格的版面边界为准：不得混入相邻图表、段落正文、页眉、页脚或孤立的残留文字。
- 每张图片或表格截图均须保留原书的图号/表号及图题/表题；裁剪前后应目视核对，既不能截断图表编号、标题、坐标轴、图例或注释，也不应为保留它们而带入正文行。
- 图表位于正文段落之间时，应优先在图题/表题之后、下一段正文之前收边；若图题位于图表上方，则应从图题上方开始裁剪。
- 替换已有截图后，应逐张打开检查边缘；发现正文残留、标题缺失、相邻资源混入或内容被裁断时，必须重新裁剪后再提交。

## 边界与保密

- 不在本目录写入 PDF 打开密码、账号、令牌或其他敏感信息。
- 整理完成后检查所有导航链接、资源路径、图表引用、代码块和公式块是否有效。
